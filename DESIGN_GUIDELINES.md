## Дизайн-гайдлайн: Fantasy-Horror с киберпанковыми элементами (единая атмосфера)

### 1. Общая концепция
- **Настроение:** тёмное фэнтези, мистика, готика, дополненные холодным свечением и технологическими деталями (киберпанк).
- **Визуальный язык:** смесь органических форм (драпировки, резьба, маски) и геометрических неоновых линий. Переходы плавные, но с резкими акцентами.
- **Цель:** погрузить пользователя в мир, где магия и технологии переплетены. Каждый товар — артефакт.
- **MVP и scope:** оформление заказа и корзина в первой версии не реализуются ([OVERVIEW.md](OVERVIEW.md)). На детальной странице первичный призыв к действию — **«Связаться» / «Уведомить о поступлении»** (или одна главная кнопка без корзины), визуально как сильный акцентный CTA (градиент и ховер из п. 4.2).

---

### 2. Цветовая система (конфигурируемая)

Цвета задаются через CSS-переменные и маппинг в `tailwind.config.js` (или эквивалент в **Tailwind v4** — сверить с фактическим конфигом после `rails new -c tailwind`). Ниже — **разделение «стиль интерфейса» и «категория товара»**, чтобы не путать кибер-акцент UI с категорией `cyberpunk` в данных.

#### Фон и базовые цвета
- `--color-bg-primary`: `#0A050F` (глубокий чёрно-фиолетовый) – основной фон
- `--color-bg-secondary`: `#120B1A` (тёмный с примесью пурпурного) – для карточек, модальных окон
- `--color-surface`: `#1E1528` (элементы интерфейса, панели)
- `--color-border`: `#2C1F3A` (границы, разделители)

#### Текст
- `--color-text-primary`: `#F0E9F0` (светлый с лёгким розоватым оттенком)
- `--color-text-secondary`: `#B9A9C9` (приглушённый лавандовый)
- `--color-text-accent`: `#E85D5D` (акцентный, для цен, кнопок)

#### Акценты интерфейса (не путать с категориями)
- `--color-accent-1`: `#8B5F9E` (основной акцент – мистический пурпур)
- `--color-accent-2`: `#5D3A6B` (темнее, для теней/глубины)
- `--color-accent-glow`: `#B76FE0` (свечение, неоновый фиолет)
- `--color-accent-cyber`: `#00C8B0` (киберпанковый бирюзовый UI — линии, иконки, рамки, навигация)
- `--color-accent-cyber-glow`: `#66FCF1` (свечение кибер-элементов интерфейса)

#### Категории товара (согласовано с `Product#category` в OVERVIEW)
Используются в фильтрах, чипсах, опционально в бейджах карточки:
- Fantasy: `--color-category-fantasy`: `#9B6BB5`
- Horror: `--color-category-horror`: `#B22234`
- Sci‑fi: `--color-category-scifi`: `#4A90D9` (холодный sci‑fi; при необходимости заменить на брендовый оттенок)
- Cyberpunk: `--color-category-cyberpunk`: `#00C8B0` (совпадает по оттенку с UI-кибером, но **семантически** это токен категории)

В основном интерфейсе преобладают `accent-1`, `accent-2`, `accent-glow`; `--color-accent-cyber*` — дозированно. Для фильтров — токены `category-*`.

### 3. Типографика

- **Заголовки (H1-H3):** *Cinzel* (serif) – для фэнтезийной основы, с увеличенным межбуквенным интервалом.
- **Подзаголовки, навигация, кнопки:** *Orbitron* (sans) – добавляет технологичность.
- **Основной текст:** *Inter* – чистый, читаемый.
- **Цифры, цены, акцентные элементы:** *JetBrains Mono* – моноширинный для кибер-акцентов.

**Размеры:**
- H1: 3.5rem (56px)
- H2: 2.25rem (36px)
- H3: 1.5rem (24px)
- Body: 1rem (16px)
- Мелкий: 0.875rem (14px)

---

### 3.1. Сетка отступов и радиусы

Базовая шкала **4px / 8px** (0.25rem шаг):
- **xs** 4px, **sm** 8px, **md** 16px, **lg** 24px, **xl** 32px — для padding/margin/gap карточек, секций, форм.

**Радиусы:**
- **Карточка товара:** `1rem` (16px)
- **Кнопки, инпуты:** `0.5rem` (8px)
- **Чипсы / мелкие контролы:** `9999px` (pill) или `0.375rem`

### 4. Компоненты и паттерны (единый стиль)

#### 4.1. Карточка товара
- **Фон:** `--color-bg-secondary`, граница `1px solid var(--color-border)`, скругление по §3.1.
- **Ховер:** подъём (`translateY(-6px)`), свечение `--color-accent-glow` (`0 0 12px`). Внутренняя подсветка по краям.
- **Изображение/3D-превью:** в карточке — статическое изображение (обложка Active Storage) или мини-рендер WebGL (опционально). Обрезка — скруглённый прямоугольник или круг по сетке.
- **Название:** Cinzel, `--color-text-primary`.
- **Цена:** JetBrains Mono, `--color-text-accent`.
- **Кнопка «Подробнее»:** прозрачная, обводка `--color-accent-1`, при ховере заливка `--color-accent-2`, текст `--color-text-primary`. Состояния кнопок — §4.6.

#### 4.2. Детальная страница товара
- **Сетка:** левая колонка (3D-превью) ~60%, правая (информация) ~40%; на tablet/mobile — колонка с canvas сверху.
- **3D-контейнер:** затемнённый фон с градиентом от `--color-bg-primary` к `--color-accent-2`, рамка со свечением `--color-accent-glow`.
- **Описание:** `--color-text-secondary`, декоративные элементы по желанию.
- **Главный CTA (MVP, без корзины):** градиент от `--color-accent-1` к `--color-accent-2`, лёгкая пульсация тени при ховере (см. §4.5); подпись кнопки по смыслу MVP — «Связаться» / «Уведомить» и т.д. ([OVERVIEW.md](OVERVIEW.md)).

#### 4.3. Навигация
- **Шапка:** фиксированная, фон `rgba(10,5,15,0.8)`, blur 12px. Нижняя граница — градиент от `--color-accent-glow` к прозрачному.
- **Логотип:** иконка маски + Cinzel.
- **Меню:** ховер — неоновая подсветка (`text-shadow: 0 0 4px var(--color-accent-glow)`). **Активная ссылка** — нижняя линия `var(--color-accent-cyber)` (UI-акцент, не обязательно категория товара).

#### 4.4. Фильтры и категории
- **Чипсы:** фон `--color-surface`, граница `--color-border`. Активный чипс — фон `--color-accent-1`, текст контрастный (белый/primary).
- Цветовая подсветка активного фильтра по категории — опционально токены `--color-category-*`.
- **Дропдаун:** тёмный стиль с лёгкой подсветкой границ.

#### 4.5. Микроанимации (CSS + Tailwind)
Численные ориентиры (уточнять в `transition-*` / `@keyframes`):
- **Fade-in + сдвиг появления:** 200–300ms, easing `cubic-bezier(0.4, 0, 0.2, 1)`; сдвиг по оси Y не более 8–12px.
- **Hover (тени, свечение):** 150–250ms, тот же easing.
- **Кнопки (масштаб 1.02):** 150ms; не комбинировать с длинным scale на больших областях (доступность).
- **Пульсация CTA (тень):** цикл 2–3s, `ease-in-out`, амплитуда тени умеренная.
- **`prefers-reduced-motion: reduce`:** отключить или заменить на мгновенное изменение opacity без сдвига; отключить пульсацию и скролл-«выплывание» (оставить простой fade или вообще без анимации).

**Загрузка 3D:** спиннер (например, SVG «череп») поверх canvas; прогресс-бар с `--color-accent-glow` ([OVERVIEW.md](OVERVIEW.md) § чеклист WebGL).

**Скролл-анимации:** реализация через **Stimulus + Intersection Observer** (класс на `rootMargin` / `threshold`); при `prefers-reduced-motion` — не подключать observer-анимации, показывать контент сразу.

#### 4.6. Состояния кнопок (все интерактивные кнопки)
- **Default:** как в макетах §4.1–4.2.
- **Hover:** усиление свечения/фона (уже заложено).
- **Active / pressed:** лёгкое уменьшение яркости или `scale(0.98)` на 100ms.
- **Focus (keyboard):** видимое кольцо `2px solid var(--color-accent-glow)` или `outline-offset: 2px`.
- **Disabled:** opacity ~0.5, `pointer-events: none`, без hover-эффектов.
- **Loading:** спиннер или текст «…», disabled-клики, сохранение минимальной высоты кнопки.

#### 4.7. Пустые состояния и формы
- **Каталог без товаров:** иллюстрация/иконка, короткий текст, ссылка в админку для авторизованного или нейтральное сообщение для гостя.
- **Админ-формы:** поля на `--color-surface`, границы `--color-border`, ошибки валидации — `--color-text-accent` или отдельный `--color-danger` при введении в дизайн-систему.

### 5. WebGL-превью (стиль и настройки)
- **Фон canvas:** прозрачный или тёмный градиент в тон странице.
- **Освещение:** ambient `0x442266`, back `0x226688`, направленный свет сверху — мистический объём.
- **Поведение:** медленный auto-rotate в покое; при взаимодействии — OrbitControls. На слабых устройствах auto-rotate можно отключать (логика в Stimulus).
- **Загрузка:** прогресс (полоска `--color-accent-glow`).
- **Ошибка / сеть / CORS:** плашка с понятным текстом, кнопка **«Повторить»**; дублирование fallback — 3D-куб или статичная обложка ([OVERVIEW.md](OVERVIEW.md)).

---

### 6. Адаптивность и производительность
- **Desktop:** две колонки на детальной, карточки 3–4 в ряд.
- **Tablet:** карточки 2–3 в ряд; детальная — canvas сверху.
- **Mobile:** карточки 1 в ряд; canvas на всю ширину; **touch** для OrbitControls.
- **WebGL:** `devicePixelRatio` ≤ 2 на desktop retina, **1** на типичных мобильных; уменьшение разрешения canvas при необходимости.
- **Жесты:** ограничить область вращения рамкой canvas, чтобы уменьшить конфликт со скроллом страницы; при необходимости — `touch-action` на контейнере.

### 7. Конфигурируемая цветовая система (Tailwind + CSS)

Конфиг ниже ориентирован на **Tailwind v3** (`tailwind.config.js`). В **Tailwind v4** пути к теме могут отличаться — перенести те же токены в актуальный формат после генерации проекта.

```js
module.exports = {
  theme: {
    extend: {
      colors: {
        bg: {
          primary: 'var(--color-bg-primary)',
          secondary: 'var(--color-bg-secondary)',
          surface: 'var(--color-surface)',
        },
        text: {
          primary: 'var(--color-text-primary)',
          secondary: 'var(--color-text-secondary)',
          accent: 'var(--color-text-accent)',
        },
        accent: {
          DEFAULT: 'var(--color-accent-1)',
          dark: 'var(--color-accent-2)',
          glow: 'var(--color-accent-glow)',
          cyber: 'var(--color-accent-cyber)',
          'cyber-glow': 'var(--color-accent-cyber-glow)',
        },
        category: {
          fantasy: 'var(--color-category-fantasy)',
          horror: 'var(--color-category-horror)',
          scifi: 'var(--color-category-scifi)',
          cyberpunk: 'var(--color-category-cyberpunk)',
        },
        border: 'var(--color-border)',
      },
      fontFamily: {
        heading: ['Cinzel', 'serif'],
        subheading: ['Orbitron', 'sans-serif'],
        body: ['Inter', 'sans-serif'],
        mono: ['JetBrains Mono', 'monospace'],
      },
    },
  },
}
```

В `application.css` или `app/assets/stylesheets/application.tailwind.css`:

```css
:root {
  --color-bg-primary: #0A050F;
  --color-bg-secondary: #120B1A;
  --color-surface: #1E1528;
  --color-border: #2C1F3A;
  --color-text-primary: #F0E9F0;
  --color-text-secondary: #B9A9C9;
  --color-text-accent: #E85D5D;
  --color-accent-1: #8B5F9E;
  --color-accent-2: #5D3A6B;
  --color-accent-glow: #B76FE0;
  --color-accent-cyber: #00C8B0;
  --color-accent-cyber-glow: #66FCF1;
  --color-category-fantasy: #9B6BB5;
  --color-category-horror: #B22234;
  --color-category-scifi: #4A90D9;
  --color-category-cyberpunk: #00C8B0;
}

@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

При необходимости сузить глобальное правило reduced-motion до отдельных классов утилит, чтобы не ломать ожидаемое поведение скриптов — тогда вынести сброс только на классы анимаций интерфейса.

### 8. Доступность и кросс-браузерность
- Контраст текста к фону не ниже **4.5:1**.
- Фокус: обводка `--color-accent-glow` (см. §4.6).
- WebGL: альтернатива — обложка изображения; сообщение при отсутствии поддержки.
- **ARIA** для контролов 3D и для кнопки «Повторить» при ошибке загрузки модели.
