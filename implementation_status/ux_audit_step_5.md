## UX-аудит (шаг 5 из `seo-audit-madmask`)

Дата: 2026-04-01

### Объект аудита

- Публичные страницы: `pages#home`, `products#index`, `products#show`, форма заявки (`inquiries#create.html.erb` фактически содержит форму).
- Общие элементы: `app/views/shared/_header.html.erb`, `app/views/shared/_footer.html.erb`.
- Эффекты: глобальный `cursor-smoke` canvas в `app/views/layouts/application.html.erb`, WebGL/галерея в `products#show`.

### Короткий вывод

- **Сильные стороны**: хорошие CTA на главной, понятная структура каталога, аккуратная доступность (skip-link, `aria-label`, фокус-обводки), продуманная галерея товара (lightbox, навигация, счётчик, крупные hit-area).
- **Главный UX-гэп (High)**: отсутствует сценарий “**оставить заявку по конкретному товару**” из карточки товара/страницы товара (с автоподстановкой названия и ссылки).
- **Риск мобильной плавности (Medium)**: глобальный `cursor-smoke` canvas присутствует на всех страницах и может ухудшать INP/энергоэффективность на слабых устройствах.

## 5.1 Навигация и CTA

### Факты по текущему коду

- **Header** уже содержит:
  - ссылку на каталог,
  - меню “Инфо” со страницами: about / delivery&payment / lead times & custom / materials & care / faq / contacts,
  - иконку VK,
  - кнопки корзины и избранного.
- **Footer** не пустой: дублирует набор инфоссылок + копирайт.
- **Каталог**:
  - фильтры по жанрам реализованы чипами,
  - счётчик результатов + текущий жанр отображаются.
- **Карточка товара в листинге**:
  - CTA: “Подробнее”, иконка “в корзину”, “в избранное”.
- **Страница товара**:
  - управляемые CTA корзины (qty +/- / убрать) + “В корзину” + “В избранное”,
  - **нет CTA “Оставить заявку по товару”**.
- **Форма заявки**:
  - хорошо оформлена (подсказки, ошибки, подтверждение), но не привязана к выбранному товару.

### Проблемы и влияние

- **High — нет “заявки по товару”**:
  - пользователь может хотеть уточнить наличие/финиш/сроки без “покупки в корзину”;
  - корзина не всегда соответствует фактическому процессу заказа (если продажи идут через подтверждение/переписку);
  - на мобильных “корзина/избранное” как единственные действия могут снижать конверсию в лид.

### Рекомендации (конкретные)

- **High — добавить CTA “Оставить заявку по товару”**:
  - на `products#show` рядом с “В корзину” / “В избранное” (как primary или secondary);
  - опционально в `products/_actions.html.erb` для карточек каталога (как компактная ссылка/кнопка).
- **High — автоподстановка в заявку**:
  - передавать `product_id`/`product_name`/`product_url` через query params в `new_inquiry_path`;
  - в форме заявки показывать “Интересует: <товар>” + автоматически вставлять в сообщение шаблон вида:
    - “Интересует товар: <название> (<URL>). Вопрос: …”
- **Medium — CTA в header**:
  - добавить заметную кнопку “Оставить заявку / Custom” в `header` (особенно полезно на внутренних страницах и на мобайле).

## 5.2 Читаемость и мобильная версия

### Факты по текущему коду

- В `application.html.erb` глобально рендерится `cursor-smoke` canvas:
  - `class="... h-screen w-screen"`, `pointer-events-none`, абсолютное позиционирование.
- На `products#show` WebGL присутствует только при attached модели и визуально “лениво” активируется при выборе 3D-слайда (это плюс).
- В галерее/кнопках соблюдаются большие зоны нажатия (`min-h-[44px]`, `touch-manipulation`), есть `focus-visible`.

### Риски и влияние

- **Medium — производительность/плавность на мобайле**:
  - глобальный canvas может грузить CPU/GPU даже когда пользователь просто читает страницу/заполняет форму;
  - эффекты могут ухудшить INP и автономность (особенно на устройствах low-end).

### Рекомендации (конкретные)

- **Medium — деградация эффектов**:
  - отключать или упрощать `cursor-smoke`:
    - на mobile (по ширине/UA — предпочтительнее ширина),
    - при `prefers-reduced-motion: reduce`,
    - при `save-data` (если используете Network Information API),
    - после первого user-idle / только после первого движения мыши (desktop).
- **Medium — контроль DPR/размера canvas**:
  - ограничивать \(devicePixelRatio\) для canvas (например, cap 1.5–2) и/или рендерить меньшим буфером.

## 5.3 Быстрый список задач (приоритет)

- **High**: CTA “Оставить заявку по товару” на `products#show` + автоподстановка.
- **Medium**: добавить CTA “Заявка/Custom” в header (и/или sticky CTA на мобайле на странице товара).
- **Medium**: adaptive/perf правила для `cursor-smoke` (mobile + reduced motion + cap DPR).
