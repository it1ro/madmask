# Контентный аудит (пункт 3 SEO‑аудита)

Основано на текущих шаблонах Rails (code-based) и структуре данных `Product`.

## Итог по состоянию сейчас

- **Главная (`pages#home`)**: сильная контентная страница — есть внятный оффер, H1/H2/H3, текстовые блоки, CTA, базовая микроразметка Organization.
- **Каталог (`products#index`)**: UI/фильтры есть, но как SEO‑страница сейчас «тонкий» — нет `meta_description`, нет уникального H1/вступления для жанровых фильтров, нет текстового блока/FAQ, который бы объяснял ценность жанра и отвечал на вопросы.
- **Карточка товара (`products#show`)**: структура хорошая (галерея/3D, хлебные крошки, цена, описание условно), но **нет `meta_description` и мета‑картинки товара**. При пустом описании блок пропадает → риск «тонкого» контента на части товаров.
- **Заявка (`inquiries#new/create`)**: страница полезная для UX и конверсии, но обычно не является SEO‑точкой роста. Сейчас в `meta robots` уже заложено `noindex` только для `/inquiries/thanks`; для самой формы индексацию нужно решить отдельно (см. ниже).

## 3.1 Уникальность / достаточность контента

### Товары

**Что хорошо**

- В модели `Product` есть требование к описанию на языке по умолчанию: минимум **200 символов** (`MIN_DESCRIPTION_CHARS = 200`). Это снижает риск совсем «пустых» карточек, если валидации реально соблюдаются при заведении товаров.

**Риски**

- 200 символов — это нижняя граница для «не пусто», но недостаточно для устойчивой релевантности и конверсии по коммерческим запросам.
- В карточке товара нет структурированного контента, кроме одного текстового блока: нет материалов/посадки/ухода/сроков/вариантов кастома, нет FAQ.

**Рекомендация (High): стандарт карточки товара**

- **Описание**: 120–250 слов (≈ 700–1600 знаков) уникального текста на RU + краткая EN‑версия при необходимости.
- **Структура (H2/секции)**:
  - «Для чего подходит» (сцена/съёмка/косплей + ограничения, если есть).
  - «Материалы и финиш» (1–3 варианта; если вариативно — описать диапазон).
  - «Посадка и крепление» (как фиксируется, комфорт/вентиляция).
  - «Наличие и сроки» (в наличии/под заказ + ориентир по срокам).
  - «Уход» (кратко: хранение/чистка/что нельзя).
  - «FAQ» (3–6 вопросов).

### Каталог

**Риски**

- Без текста/FAQ жанровые фильтры выглядят как «технические» страницы (листы), слабая релевантность по запросам «маска horror купить/заказать» и т.п.

**Рекомендация (High): сделать жанровые страницы «полноценными»**

При выбранном `category` добавить:

- 1–2 абзаца (300–600 знаков) под H1: что за жанр, чем отличается коллекция, куда подходит.
- мини‑FAQ (3–5 вопросов).
- блок перелинковки (подкатегории/похожие жанры/ссылки на «custom»).

## 3.2 Заголовки H1–H6

### Главная

- H1 есть и ок (позиционирование, короткое).
- H2/H3 есть и работают как «сканируемая» структура.

### Каталог

**Проблема (Medium)**: H1 всегда «Каталог», даже при фильтре жанра.

**Рекомендация (Medium/High)**:

- Если жанровые фильтры индексируем: H1 должен быть **уникальным по жанру**.
  - Пример: «Маски Horror — Madmask» / «Fantasy маски ручной работы — Madmask».
- Если жанровые фильтры не индексируем: можно оставить H1 «Каталог», но тогда:
  - каноникал на `/products`,
  - а фильтры использовать как UX‑функцию (без попытки ранжировать).

### Товар

- H1 — название товара, ок.
- Не хватает дополнительных секций (см. 3.1) → добавятся H2/H3.

## 3.3 Title / Description (meta)

### Что сейчас

- **Home**: и `title`, и `meta_description` заданы.
- **Catalog**: `title` задан, `meta_description` отсутствует.
- **Product**: `title` задан, `meta_description` отсутствует.
- **Inquiry**: в текущем шаблоне формы не задано `content_for :title / :meta_description` (страница будет с дефолтным `Madmask`).

### Рекомендации (High): шаблоны meta description

- **Каталог общий**: 140–160 символов. Упор на жанры, ручную работу, доставку, возможность custom.
  - Пример: «Каталог Madmask: маски fantasy, horror, sci‑fi и cyberpunk. Ручная сборка, малые серии, доставка по РФ. Открой карточку, осмотри детали и оставь заявку.»
- **Каталог по жанру** (если индексируем):
  - Пример: «Маски Horror от Madmask: малые серии и custom под референсы. Для сцены, съёмок и косплея. Осмотри детали товара и оставь заявку.»
- **Товар**:
  - Формула: «<Название> — маска <жанр>. <1 фраза из описания>. Для сцены/съёмки/косплея. Оставь заявку — уточним наличие и сроки.»
  - Источник: первые \(150–170\) символов из `description` + безопасная обрезка по словам.
- **Форма заявки** (если оставляем index — спорно):
  - «Оставь заявку на маску Madmask: опиши задачу, приложи референсы. Ответим по материалам, срокам и наличию обычно в течение дня.»

### Рекомендации (High): meta image для товара

- Для `products#show`: `meta_image` = обложка товара (или первый кадр галереи).
- Это улучшит превью в соцсетях и CTR при шаринге.

## 3.4 Ключевые слова (встраивание семантики)

**Цель**: не «набивать», а естественно встроить смысловые маркеры туда, где их ждут и люди, и поисковик.

- **Каталог**:
  - H1 + 1 абзац: «маски <жанр>», «ручная работа», «для сцены/съёмок/косплея».
  - Анкоры: ссылки «custom маска / на заказ», «сроки изготовления», «материалы и уход».
- **Товар**:
  - 1 абзац под ценой: 1–2 ключевые фразы (жанр + сценарий использования).
  - FAQ: вопросы часто совпадают с хвостами запросов («как крепится», «можно ли по референсу», «сроки», «как ухаживать»).
- **Изображения**:
  - `alt` уже неплохой (название товара / описание hero). Для галереи миниатюр `alt` пустой — это нормально для декоративных превью, но для главного изображения alt должен быть описательным (он уже = название товара).

## 3.5 E‑E‑A‑T (доверие и экспертность)

**Что есть**

- Organization JSON‑LD (минимальный) + ссылка на VK.
- Инфостраницы уже заведены в навигации (`about`, `delivery_and_payment`, `lead_times_and_custom`, `materials_and_care`, `faq`, `contacts`) и есть шаблон `pages/_info_page.html.erb` для структурированного контента.

**Пробелы (High)**

- На уровне **карточек товара** не хватает доверительных деталей: материалы, уход, сроки, условия custom, ответы на вопросы.
- На уровне **каталога/жанров** — нет объяснения коллекций.

**Рекомендации (High)**

- На каждой карточке товара добавить «прозрачные» блоки: материалы/посадка/уход/сроки/FAQ.
- На страницах инфо (если ещё не наполнены) — добавить фото/описание процесса, условия доставки/оплаты, сценарии кастома, контактные каналы.

## Решение по индексации страницы «Заявка»

Есть два жизнеспособных варианта (выберите один и закрепите в мета/robots):

- **Вариант A (рекомендуемый по умолчанию)**: `inquiries#new` → `noindex, follow`.
  - Логика: форма — конверсионная страница, но редко нужна в поиске; лучше продвигать «Custom / на заказ» как отдельный лендинг с контентом, а форму оставить как шаг.
- **Вариант B**: оставить index, но сделать это полноценным «Custom / на заказ» лендингом.
  - Логика: тогда это не просто форма, а страница с контентом (процесс, примеры, сроки, FAQ) + форма внизу.

## Следующие внедрения (после аудита)

- Добавить `meta_description` для `products#index` и `products#show`.
- Сделать жанровый H1 + текст/FAQ на каталоге при фильтре.
- Добавить `meta_image` для `products#show` (обложка/первое изображение).
- Решить стратегию индексации `inquiries#new` и при необходимости добавить `noindex`.
