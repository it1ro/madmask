## Выполнено: пункт 3.1 (Контентный аудит → Уникальность/достаточность)

### Цель
Снизить риск «тонкого контента» на карточках товаров из-за пустых/минимальных описаний.

### Что сделано
- **Валидация контента товара**: в `Product` добавлено требование к описанию на языке по умолчанию (RU) — описание обязательно, минимум 200 символов.
- **Подсказка в админке**: в форме товара добавлен шаблон/структура для описания (что это, для чего подходит, материалы/уход, сроки/наличие, мини‑FAQ) и явная подсказка по минимальному объёму.

### Затронутые файлы
- `app/models/product.rb`
- `app/views/admin/products/_form.html.erb`

### Примечания
- Если в базе уже есть товары без RU-описания, их нужно заполнить при следующем редактировании/сохранении товара в админке (иначе валидация не пропустит сохранение).
