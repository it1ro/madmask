# Итерация 10.2: Товар — 3D как часть галереи (рефакторинг)

| Подзадача | Статус |
|-----------|--------|
| Встроить 3D превью в существующую галерею на `products#show` | выполнено |
| Если `model_file` доступен — показывать 3D по умолчанию | выполнено |
| Лайтбокс оставить только для фото (3D не открывает fullscreen) | выполнено |
| Сохранить остальной UX галереи (миниатюры/счётчики/навигация) | выполнено |

## Что изменили (факт в коде)

- **Единый блок галереи** на детальной странице товара:
  - 3D больше не отдельной секцией ниже — теперь это первый «слайд» внутри галереи: `app/views/products/show.html.erb`.
- **Типизированные слайды**:
  - `slidesValue` теперь содержит элементы вида:
    - `{ type: "webgl", modelUrl, coverImageUrl }`
    - `{ type: "image", src, srcset }`
  - 3D-слайд добавляется первым, только если прикреплён `model_file`: `app/views/products/show.html.erb`.
- **Lazy инициализация Three.js**:
  - `webgl-preview` активируется только когда пользователь находится на 3D-слайде (по умолчанию при наличии модели): `app/javascript/controllers/product_gallery_controller.js`.
- **Лайтбокс только для фото**:
  - `openLightbox` игнорируется на `webgl`-слайде; навигация лайтбокса пролистывает только `image`-слайды: `app/javascript/controllers/product_gallery_controller.js`.

## Критерии проверки

### Товар с `model_file`
- При открытии страницы **сразу виден 3D** в галерее.
- В миниатюрах первым пунктом есть **“3D”**, дальше — фото.
- Клик по 3D области **не открывает** fullscreen.
- Переключение на фото работает как раньше; fullscreen открывается только для фото.

### Товар без `model_file`
- Галерея работает как раньше (только фото).

## Примечания

- В лайтбоксе счётчик показывает количество **фото**, а не общее число слайдов (3D туда не входит).
