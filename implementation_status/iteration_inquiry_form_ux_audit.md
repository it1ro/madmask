## Итог работ: форма заявки (`inquiries#create` view)

### Цели
- Сделать отправку заявки понятнее (микрокопирайт + ожидания по срокам).
- Улучшить доступность и UX ошибок (summary + ошибки у полей).
- Подтянуть клиентские ограничения под серверные валидации.
- Добавить базовую антиспам защиту.

### Сделано
- **Микрокопирайт**: единый тон обращения на странице и в flash (informal), добавлен ориентир по срокам ответа.
- **A11y/ошибки**:
  - summary ошибок теперь `role="alert"` + `aria-live="polite"`;
  - ошибки показываются рядом с соответствующим полем;
  - добавлены `aria-invalid` и `aria-describedby` (для email — с подсказкой + ошибкой);
  - фокус на первом проблемном поле через `autofocus` при повторном рендере с 422.
- **Клиентские ограничения**: добавлены `maxlength` в соответствии с `Inquiry` валидациями.
- **Антиспам (honeypot)**: скрытое поле `website` в форме + проверка в контроллере.
- **Нормализация ввода**: `strip`/`presence` для полей перед валидациями.

### Изменённые файлы
- `app/views/inquiries/create.html.erb`
- `app/controllers/inquiries_controller.rb`
- `app/models/inquiry.rb`

### Примечание
- Плановое приведение view к конвенции (`new.html.erb` + `_form.html.erb`) не выполнено из-за ограничений на создание новых файлов в `app/views/inquiries` в текущем окружении.
