## Цель
- Продолжить ускорение загрузки и рендеринга без запуска нагрузочных тестов.
- Снизить клиентскую нагрузку на первый экран за счет менее агрессивной инициализации JS и эффектов.

## Изменения

### 1) Lazy загрузка Stimulus-контроллеров
- Файл: `app/javascript/controllers/index.js`
- Было: `eagerLoadControllersFrom("controllers", application)`
- Стало: `lazyLoadControllersFrom("controllers", application)`

Эффект:
- Контроллеры подгружаются по факту появления `data-controller` в DOM, а не все сразу.
- Уменьшается стартовая JS-нагрузка на первом экране.

### 2) Cursor-smoke canvas только на главной
- Файл: `app/views/layouts/application.html.erb`
- Canvas с `data-controller="cursor-smoke"` теперь рендерится только для `pages#home`.

Эффект:
- На остальных страницах убран лишний canvas-слой и исключена попытка инициализации эффекта.
- Меньше лишней работы браузера вне лендинга.

### 3) Более легкий blur в шапке на mobile
- Файл: `app/views/shared/_header.html.erb`
- Было: `backdrop-blur-[12px]`
- Стало: `backdrop-blur-[6px] md:backdrop-blur-[12px]`

Эффект:
- Снижается стоимость композитинга на мобильных устройствах.
- Визуальная стилистика остается, но фильтр дешевле на узких экранах.

## Проверка (без нагрузочных тестов)
- Выполнена проверка ответа `GET /` после правок.
- Проверены линтеры на измененных файлах: ошибок не найдено.

## Примечание
- Этот этап сознательно выполнен без `bin/load_test_ab` / `bin/load_test_latencies` по запросу.
- Для следующего шага можно сделать только браузерный профайлинг (Lighthouse/Performance) без серверного load-testing.
